# Abstract Data Types (ADT)

## Definitions

| Term | Definition |
|------|-----------|
| **DATA** | Raw facts |
| **STRUCTURE** | Way of organizing data in memory |
| **DATA STRUCTURE** | Processing information by organization of data |
| **Algorithm** | Step-by-step organization of data used to solve a problem |

### Applications of Data Structures
- Ticket booking systems
- World Trees
- Order stacking

---

## Types of Data

1. **Primitive** — Basic data types provided by the programming language
2. **Non-Primitive** — Derived from primitive data types; used to store multiple values
3. **Linear** — Elements stored in a linear/sequential manner
4. **Non-Linear** — Elements arranged hierarchically

---

## Abstract Data Type (ADT)

> An ADT is a **logical description** of data and operations on the data **without specifying implementation details**.

### ADT vs Data Structure

| ADT | Data Structure |
|-----|----------------|
| Logical view | Physical implementation of data |
| Language independent | Language dependent |
| What to do | How to do |
| It defines abstraction | Multiple subtypes available |

### Advantages of ADT
1. **Abstraction** — hides implementation details
2. **Simple & easy to use**
3. **Improves reusability**
4. **Provides modularity**
5. Arrays use less time (better performance)

---

## Common ADT Examples

| Stack ADT | Queue ADT | List ADT |
|-----------|-----------|----------|
| `Push()` | `Enqueue()` | `Insert()` |
| `Pop()` | `Dequeue()` | `Search()` |
| `Peek()` | — | `Delete()` |

### Array as ADT
- Similar data type
- Fixed size
- Contiguous memory locations
